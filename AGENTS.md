# Agent Guidelines

## Git Commits

- When a task is complete and it makes sense to persist the work, create an atomic commit for that task.
- Stage only the files changed for that specific task. Do not include unrelated local changes.
- Use a clear, appropriate commit message that describes the completed task.
- If a task naturally splits into independent changes, use separate atomic commits for each change.
- Automatically commit after every single logical change without being asked. Do not wait for the user to prompt you.
- Do not commit work that is unfinished, unverified, or mixed with unrelated edits unless the user explicitly asks for it.
